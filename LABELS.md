# Issue / PR ラベル運用

このリポジトリでは、Issue / PR を整理しやすくし、
**環境構築段階〜実装フェーズまで一貫した運用**を行うため、
以下のルールでラベルを使用する。

---

## 🎯 運用方針（重要）

* **ラベルは役割ごとに分類する**
* **各カテゴリのラベルは原則1つまで**
* 迷ったら「type → area → prio → status」の順で付ける
* 新しいラベルを追加する前に、既存ラベルで代替できないか検討する

---

## 🧩 ラベルカテゴリ一覧

### 1️⃣ type（作業の種類）

> **必須（原則1つ）**

| ラベル            | 説明                              |
| -------------- | ------------------------------- |
| `type:setup`   | 開発環境構築、初期設定、ツール導入               |
| `type:docs`    | README / SETUP / USAGE などドキュメント |
| `type:chore`   | 雑務、リファクタ、設定整理                   |
| `type:bug`     | 不具合修正                           |
| `type:feature` | 新機能追加                           |

---

### 2️⃣ area（対象領域）

> **1〜2個まで**

| ラベル            | 説明                  |
| -------------- | ------------------- |
| `area:core`    | コアロジック              |
| `area:cli`     | CLI / 実行ファイル        |
| `area:scripts` | 補助スクリプト             |
| `area:ci`      | CI / GitHub Actions |
| `area:docs`    | ドキュメント              |

---

### 3️⃣ prio（優先度）

> **必要な場合のみ（最大1つ）**

| ラベル       | 説明          |
| --------- | ----------- |
| `prio:p0` | 最優先（作業が止まる） |
| `prio:p1` | 高           |
| `prio:p2` | 通常          |

---

### 4️⃣ status（状態）

> **詰まりポイントの可視化用**

| ラベル                 | 説明                   |
| ------------------- | -------------------- |
| `status:blocked`    | 他Issue・外部要因待ちで進められない |
| `status:needs-info` | 情報不足・調査待ち            |

---

## 🧠 ラベル付与ルールまとめ

* `type`：**必ず1つ**
* `area`：**1〜2つ**
* `prio`：**必要なら1つ**
* `status`：**状態を表す場合のみ**

❌ 同じカテゴリのラベルを複数付けない
❌ 似た意味のラベルを増やさない

---

## 📌 例

### 環境構築タスク

```
type:setup
area:ci
prio:p0
```

### README 整備

```
type:docs
area:docs
```

### 調査待ちのIssue

```
type:chore
status:needs-info
```

### 外部対応待ちで止まっているIssue

```
type:setup
status:blocked
```

---

## 🚦 環境構築フェーズでの考え方

* 実装前は **`type:setup` / `type:docs` / `type:chore` が中心**
* `type:feature` は「実装フェーズ突入の合図」
* `status:*` を使って「止まっている理由」を明確にする

---

## 🔧 ラベル追加・変更ルール

* 新しいラベルが必要になった場合は **Discussion で提案**
* 既存ラベルとの重複がないことを確認する
* 採用後、このドキュメントを更新する
